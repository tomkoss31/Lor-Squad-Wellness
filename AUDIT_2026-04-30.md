#### `src/components/client-detail/SportSummarySection.tsx`

#### `src/lib/portfolio.ts`

**Fix** : reformaté en single-statement avec `[0]` collé.

### 2. ✅ Bug "Edge Function returned a non-2xx status code" — 4 flows critiques → fixés

**Cause racine** : Supabase-js v2.101+ a changé le comportement de `functions.invoke()`. Quand l'Edge function renvoie 4xx/5xx, `data` est `null` et le body JSON (qui contient le vrai message d'erreur) n'est plus accessible via `data.error`. Il est dans `error.context.response` (objet Response qu'il faut `clone().json()`).

**Impact** : tous les flows d'edge function affichaient un message générique inutile au lieu du vrai message backend (ex: Sabrina vue chez Mélanie ce matin).

**Fix** :
- ✅ Nouveau helper `src/lib/utils/extractFunctionError.ts` qui extrait le 1er message dispo (data.error → body.error → body.message → error.message → fallback).
- ✅ Appliqué sur les 4 flows user-facing critiques :
  - `BienvenuePage` (signup client)
  - `BienvenueDistriPage` (signup distri)
  - `AutoLoginPage` (magic link)
  - `ProspectFormModal` (lead public Welcome)

**À faire (V2)** : appliquer aux 6 autres flows non critiques (`MagicLinkFallback`, `SharePublicButton`, `InviteDistributorModal`, `SharePage`, `useAutoNotifications`, `DebugNotificationsPage`).
